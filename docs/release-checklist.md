# Release checklist

## 1. Scope
- [ ] Для релиза выбран понятный scope
- [ ] Все не-блокирующие идеи вынесены из текущего релиза в backlog
- [ ] Для каждого закрываемого тикета есть понятный критерий готовности

## 2. Stability
- [ ] Локально проходит `npm run check`
- [ ] GitHub Actions проходит на последнем коммите
- [ ] Нет известных критических багов в auth / сменах / выплатах / импорте-экспорте

## 3. Manual QA smoke
- [ ] Вход / выход из аккаунта
- [ ] Создание сотрудника
- [ ] Архивация сотрудника
- [ ] Изменение ставки
- [ ] Отметка смены в календаре
- [ ] Добавление выплаты
- [ ] Подтверждение выплаты
- [ ] Удаление выплаты
- [ ] Экспорт расчетного листа
- [ ] Экспорт резервной копии
- [ ] Импорт резервной копии

## 4. Data / security
- [ ] Проверены сценарии owner и employee
- [ ] Изменения схемы БД применены и проверены
- [ ] Нет утечки чужих данных между пользователями

## 5. Release notes
- [ ] Список изменений собран коротко и по делу
- [ ] Известные ограничения записаны отдельно
- [ ] Версия или дата релиза зафиксирована

## 6. After release
- [ ] Открыты тикеты на баги и улучшения, которые не вошли в релиз
- [ ] Следующий релиз имеет черновой scope
